# 🐱 基于声音与行为分析的"喵语"情感识别交互界面开发方案

## 📋 项目概述

本方案基于开题报告《基于声音与行为分析的"喵语"情感识别与需求解读系统》第五阶段"成果集成与展示"的要求，开发一个交互式网页界面，实现上传猫咪图片/音频后自动分析并显示识别结果的功能。

**核心特点：**
- 支持图片（姿态识别）和音频（声音分类）双模态输入
- 多模态融合综合判断猫咪状态
- 采用5种标准状态标签
- 学生友好的技术实现方式

## 🔧 技术方案

### 平台选择策略

考虑到开题报告推荐使用适合青少年的AI开发平台，本方案提供两种技术路线：

#### 路线A：百度EasyDL（推荐）

**适用场景：** 已在EasyDL训练好模型并部署为API服务

- **图片识别**：调用EasyDL图像分类API
- **音频识别**：调用EasyDL语音识别API或上传音频文件
- **融合逻辑**：前端实现多模态融合算法

**优点**：无需处理模型文件，调用API即可
**缺点**：需要联网，API调用有次数限制

#### 路线B：Google Teachable Machine

**适用场景：** 使用Teachable Machine训练模型

- **图片识别**：导出TensorFlow.js模型，前端加载
- **音频识别**：导出TensorFlow.js模型或使用浏览器原生音频API
- **融合逻辑**：前端实现多模态融合算法

**优点**：纯前端运行，无需服务器
**缺点**：模型文件较大，加载较慢

### 多模态融合算法

核心逻辑：结合图片识别和音频识别的结果，给出最终判断

```
输入：图片识别结果（概率分布）+ 音频识别结果（概率分布）
权重：图片权重 W_img = 0.5，音频权重 W_aud = 0.5
综合得分：Score_i = W_img × Prob_img_i + W_aud × Prob_aud_i
最终结果：选择得分最高的状态标签
```

**权重可调整策略：**
- 如果只有图片输入：W_img = 1.0, W_aud = 0.0
- 如果只有音频输入：W_img = 0.0, W_aud = 1.0
- 如果两者都有：默认各占50%

## 📝 核心代码逻辑

### 多模态融合算法（fusion.js）

```javascript
// 状态标签映射
const STATE_LABELS = {
  1: '饥饿乞食',
  2: '撒娇求关注',
  3: '警惕警告',
  4: '愉悦呼噜',
  5: '无聊求玩'
};

// 融合函数
function fuseResults(imageResult, audioResult) {
  const weights = {
    image: imageResult ? 0.5 : 0,
    audio: audioResult ? 0.5 : 0
  };
  
  const scores = [0, 0, 0, 0, 0];
  
  if (imageResult) {
    imageResult.probabilities.forEach((prob, idx) => {
      scores[idx] += prob * weights.image;
    });
  }
  
  if (audioResult) {
    audioResult.probabilities.forEach((prob, idx) => {
      scores[idx] += prob * weights.audio;
    });
  }
  
  const maxIdx = scores.indexOf(Math.max(...scores));
  return {
    stateId: maxIdx + 1,
    stateName: STATE_LABELS[maxIdx + 1],
    confidence: scores[maxIdx],
    scores: scores
  };
}
```

#### 路线A（EasyDL）：
1. 注册百度AI开放平台账号
2. 创建图像分类应用，获取API Key和Secret Key
3. 实现图片上传和Base64编码
4. 调用EasyDL图像分类API
5. 解析返回结果并展示

1. 将训练好的模型导出为TensorFlow.js格式
2. 实现模型加载逻辑
3. 实现图片预处理（resize、归一化）
4. 调用模型进行推理
5. 解析结果并展示

### 阶段3：音频识别功能开发（2天）

#### 路线A（EasyDL）：
1. 创建语音识别应用
2. 实现音频录制功能（Web Audio API）

4. 调用EasyDL语音分类API
5. 解析返回结果并展示

#### 路线B（Teachable Machine）：
1. 将音频模型导出为TensorFlow.js格式
2. 实现音频录制和特征提取
3. 调用模型进行推理
4. 解析结果并展示

### 阶段5：可乐&草草专属功能（1天）

1. 实现快速识别入口
2. 实现历史记录功能（LocalStorage）
3. 添加统计信息展示
